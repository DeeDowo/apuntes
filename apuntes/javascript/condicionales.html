Condicionales

En cualquier lenguaje de programación se necesitan tomar decisiones para ejecutar acciones. las condicionales permiten representar esta toma de decisiones.

If...else
El tipo más básico de condicional, el if...else. "if" es la palabra clave, seguida de un par de paréntesis que toma como argumento alguna expresión y cuando esta sea cierta o se pueda entender como cierta (true), se ejecutará el bloque de código, si este argumento no es verdadero, entonces se ejecuta el bloque contiguo a la palabra clave "else". Esencialmente se toma un parámetro que dependiendo de valor booleano (true/false), hará que se ejecute un bloque de código u otro. La parte de "else" no es realmente necesaria, ya que esta solo indica el caso contrario y que en caso de no haberlo, no tendría sentido agregarlo.
Else if
Con "if...else" tenemos la posibilidad de tener dos posibles resultados para cuando una expresión es "true" o "false". Para poder tener más de estar dos posibilidades existe la palabra clave "else if", la cual se "encadena" como un "else" normal, con la diferencia de que este incluye su propia expresión de control. Es posible encadenar multiples "else if".
Al utilizar multiples "else if" podemos agregar un "else" final, el cual ejecutará su código cuando ninguna de las condiciones anteriores hayan sido verdaderas. Nuevamente, este no es necesario.

Algo a considerar es que cualquier valor que no sea "false", "undefined" (no existe la variable), "null", "NaN" (not a number) o un string vació (""), devuelven "true" al ser utilizados como una condición. De esta forma se puede usar como condición una variable y si existe y es valida, devolverá true.

Anidando if...else

Es posible anidar un "if...else" dentro de un "if..else", cada uno de estos, funcionan de manera independiente, sin embargo, anidar demasiados pueden complicar el código y hacerlo difícil de comprender.

Operadores lógicos

Para poder crear "test" más complejos podemos utilizar los operadores lógicos, esta es una alternativa a usar "if...else" anidados. Existen tres:

AND
El operador "AND" (&&) compara si ambos expresiones son "true", si lo son devuelve "true", si no lo son devuelve "false".
OR
El operador "OR" (||) compara dos expresiones y si al menos una es "true" devuelve "true" como resultado.
NOT
El operador "NOT" (!), no compara realmente, este "niega" una expresión, es decir, devuelve el valor contrario al original. En otras palabras, "NOT" (!) invierte el valor original

Los diferentes operadores pueden combinarse en cadenas con cualquier complejidad para crear expresiones más complejas.

Switch

A simple vista parece similar a "if...else", sin embargo, el "switch" se especializa en comparaciones de valores, es decir, tomando un valor como referencia, compara este con diferentes casos predefinidos y si este valor es igual al de alguno de los casos, realiza las acciones especificadas. Otra forma de entenderlo, es que "if...else" suele servir más para casos más "generales" o "flexibles", mientras que switch sirve más para casos "particulares" o "rígidos". la estructura de un "switch" es:

En este ejemplo tenemos:

  1. La palabra clave "switch".
  2. Un par de paréntesis que envuelven la expresión o variable que vamos a comparar con los diferentes casos disponibles.
  3. La palabra clave "case" para indicar cada caso posible.
  4. Un valor para cada caso.
  5. Un código que se ejecutará cuando el valor del caso y el valor a comparar sean iguales.
  6. La palabra clave "break" al final de cada caso para indicar que el "switch" terminó, de esta forma no revisará ninguno de los siguientes casos posibles y solo se "saldrá" del "switch".
  7. Cuando ninguno de los casos coinciden, es recomendable tener un caso especial, este se define con la palabra clave "default", no posee un valor e indica el código que se ejecutará cuando ningún caso coincida con el valor a comparar ingresado. En el caso "default" no es necesario indicar un "break" ya que al ser el ultimo caso, de cualquier forma el "switch statement" finalizará.

Operador ternario

El operador ternario, podría entenderse como una contracción de un "if...else", ya que toma una condición y con base en el resultado booleano de esta, ejecuta un código u otro, generalmente se usa para asignar valores a variables.
