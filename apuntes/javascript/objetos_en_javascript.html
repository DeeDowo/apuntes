Dot notation

La notación para acceder a las propiedades de un objeto o a sus métodos es llamada "dot notation", esta hace referencia a agregar un "." entre el nombre del objeto y la propiedad o método. El nombre del objeto es indispensable para poder acceder a la información dentro de este.

Objetos como propiedades

Una propiedad de un objeto puede ser a su vez otro objeto

Para acceder a estas propiedades anidadas, solo hay que encadenar los multiples objetos de la siguiente forma:

  • persona.nombre.nombre1;
  • persona.nombre.nombre2;

Bracket notation

Esta es una forma alternativa a la "dot notation" para acceder a las propiedades y métodos de los objetos.

  • persona["edad"];
  • persona["nombre"]["nombre1"];

Esta notación es bastante similar a la forma en que se ingresa a un elemento de un array, en lugar de usar un indice, se utiliza el nombre de la propiedad o método envuelto entre comillas, por esta similitud es que los objetos pueden ser llamados "associative arrays".

Qué notación utilizar

La "dot notation" sigue siendo preferida sobre la "bracket notation" ya que es más simple y fácil de leer.

Sin embargo, existen situaciones donde no es posible utilizar la "dot notation", un ejemplo es cuando el nombre de una propiedad es guardada en una variable con un nombre diferente a este, si utilizamos la "dot notation" con el nombre de esta variable, se buscará una propiedad con este nombre literal y no el del valor guardado en la variable, pero si utilizamos la "bracket notation", notaremos que funciona sin ningún problema, ya que que esta sí utiliza el valor guardado en la variable.

  • persona.variable; (buscará una propiedad en el objeto persona con el nombre "variable")
  • persona[variable]; (buscará una propiedad en el objeto persona con el nombre guardado en la variable)

Podemos crear n cantidad de objetos con esta plantilla. esto funciona bien, pero hay una forma más sencilla, esta es usando un "constructor". Los "constructors" son solo una función a la cual llamamos utilizando la palabra clave "new", cuando llamamos a un "constructor" sucede lo siguiente:

  1. Se crea un nuevo objeto temporal.
  2. Los "this" en el constructor son "reemplazados" por el objeto temporal.
  3. Una vez finalizado todo el proceso del constructor, este "objTemp" es devuelto y asignado a la variable donde lo hayamos inicializado.

La palabra clave "this" al usarse dentro un objeto hace referencia a miembros en el mismo objeto.
