Loops

La principal utilidad de los loops es repetir una misma tarea un cierta cantidad de veces o de forma indefinida. Por ejemplo, los loops para recorrer arrays ("for...of", map(), filter()).

For...of
El loop más básico para recorrer arrays en JavaScript.
Map()
Toma como argumento alguna función para realizar una cierta acción en todos los elementos de una colección y luego guardar estos en una nueva colección con los elementos modificados.
Filter()
Similar a "map()", con la excepción de que "filter()" solo crea una nueva colección con los elementos que cumplan una o varias condiciones. Esta función "test" devuelve un boolean una vez ejecutada, si el resultado el "true" el elemento se guarda en la nueva colección, si el resultado es "false", este se omite. Para poder conservar la colección resultante, hay que asignar esta a una variable

For loop

Un "for" loop, como cualquier otro loop, nos permite realizar alguna acción un número determinado o indeterminado número de veces. Este tiene la siguiente estructura:

En la estructura de un "for" loop tenemos:

  • La palabra clave "for".
  • Un par de paréntesis para indicar los parámetros de nuestro loop.
  • El "inicializador", el cual es el lugar reservado para declarar nuestra variable de control del loop, es decir, aquella que le indicará cuándo parar a este.
  • La "condición", el cual es el lugar reservado para insertar una condición, es decir, mientras esta sea verdadera, nuestro loop continuará ejecutándose. Esta condición es referente a nuestra variable de control generalmente.
  • La "expresión final", el cual es el lugar reservado en donde colocamos una expresión que modificará nuestra variable de control con cada iteración del loop y de esta forma aseguramos que en algún punto nuestro loop se terminará.
  • Por último un par de llaves que envuelven a todo nuestro código a iterar "n" número de veces.

While y do while

"for" no es el único bucle disponible en JavaScript, existen varios más, principalmente "while" y "do...while", estos 2 son bastante similares, ahí el que tengan nombres tan parecidos.

Podemos notar que la estructura de un bucle "for" y la de un bucle "while" son bastante similares, pero con un orden diferente. En el bucle "while" el orden es:

  • Inicializador.
  • Palabra clave "while".
  • Condición.
  • "Expresión final" dentro del propio código a ejecutar en el bucle.

Cada uno de los elementos cumple la misma función que en el bucle "for".

Por otra parte, la única verdadera diferencia entre un bucle "while" y un bucle "do...while" es que el bucle "do...while", siempre hace al menos una iteración del código, su estructura es:

En su estructura tenemos:

  • Inicializador.
  • Palabra clave "do".
  • "Expresión final" dentro del propio código a ejecutar en el bucle.
  • Palabra clave "while".
  • Condición.

Algo a considerar es que al trabajar con loops, es posible caer en un loop infinito, esto sucede cuando la condición expresada siempre es "true" y esto puede provocar múltiples errores. Por eso siempre hay que crear una relación sólida entre la variable de control, la expresión para modificar esta y la condición, ya que de estas tres es de quienes depende el buen funcionamiento de un bucle.

Saliendo de los loops

Cuando queremos terminar un loop antes de haber hecho todas las iteraciones o en el caso de una colección, antes de haberla recorrido por completo, podemos usar la palabra clave "break". El funcionamiento de "break" en un loop es el mismo que en un "switch statements", es decir, finaliza el proceso, en este caso, el loop. Esto es especialmente útil cuando, por ejemplo, queremos buscar un elemento en una colección y que al encontrarlo podamos salir de este loop en lugar de recorrer todos los elementos dentro de este.

Omitiendo iteraciones

Cuando queremos omitir una iteración en un loop e ir con el siguiente valor, podemos usar la palabra clave "continue", de esta forma el valor actual se descartará para la aplicación del código subsecuente a este e iremos directamente al siguiente valor como si hubiéramos terminado todo el proceso anterior. Esto es especialmente útil cuando, por ejemplo, queremos solo tomar ciertos valores que cumplan ciertas características y que aunque descartemos algún valor, sí nos interesa seguir iterando.

Qué loop utilizar?

Al iterar sobre una colección, generalmente el "for...of" es la mejor opción. Mientras que para "for", "while" y "do...while", estos generalmente pueden ser intercambiables.
