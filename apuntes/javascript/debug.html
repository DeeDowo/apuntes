Debugging JavaScript

Al trabajar con código JS es posible que nos enfrentemos a diversos errores, los maś comunes son errores de sintaxis y errores de lógica.

Para los errores de sintaxis, la propia consola nos indicará en dónde se encuentran estos. Una buena practica siempre es el verificar que nuestro código sea valido en primer lugar, esto se puede hacer comprobándolo por medio de un verificador, estos existen tanto en forma web como en forma de extensiones en algunos editores de código.

Algunos problemas comunes en JavaScript son:

  • Problemas de sintaxis o lógica.
  • Variables no definidas en el scope correcto.
  • Confusión con la palabra clave "this".
  • Uso incorrecto de loops.
  • Querer utilizar resultados de funciones asíncronas antes de tiempo.

La consola del navegador

las "devtools" de un navegador son especialmente útiles a la hora de que querer resolver errores en un código JavaScript, ya que para empezar, los errores serán reportados en la consola del navegador.

La "console API"

Esta API es la que permite interactuar al código JS con la consola de los navegadores. Como cualquier otra API esta tiene multiples funciones disponibles, algunas de las más comunes son:

console.log()
Muestra un mensaje.
console.error()
Muestra un error en rojo.
console.warn()
Muestra una advertencia en amarillo.
console.info()
Similar a "console.log", pero muestra la información de forma estilizada.
console.table()
Muestra la información en formato tabular.
console.clear()
Limpia la consola de mensajes anteriores.
console.time() - console.timeEnd()
Mide el tiempo de ejecución entre llamadas.
console.assert()
Toma 2 parámetros, una condición y un mensaje, si la condición es falsa, se muestra el mensaje.
console.group() - console.groupEnd()
Envuelve mensajes relacionados.
console.count()
Lleva el conteo de las cantidad de veces que se ha utilizado.

Es una buena practica utilizar estas funciones a lo largo de nuestro código para poder revisar el estado de nuestros programas. De todas los métodos mencionados anteriormente, los más comunes de usar son:

  • console.log()
  • console.error()
  • console.table()

Siempre se deben de eliminar todos estos mensajes una vez comprobado el correcto funcionamiento del programa.

Manejando errores en JavaScript

HTML y CSS son lenguajes permisivos, es decir, que los errores en estos generalmente solo serán ignorados y no tendrán un gran impacto en nuestras páginas. Sin embargo, javascript no es así, cuando en un código javascript se encuentran problemas, este lanzará errores.

Nuestra primera herramienta para poder manejar errores es por medio de las condicionales, de esta forma podemos ejecutar diferentes bloques de código dependiendo de diferentes resultado como posibles errores.

Errores personalizados

Es posible crear nuestros propios mensaje de error para tener aun más control sobre estos. esto se realiza creando un nuevo objeto "Error" y con la palabra clave "throw", esto "arrojará" un objeto de tipo "error" para ser manejado.
