Introducción a JavaScript

JavaScript es un lenguaje de programación que permite implementar funciones complejas en una página web. Se usa en conjunto con HTML y CSS principalmente.

HTML
Es el lenguaje de marcado que usamos para estructurar y dar significado al contenido web.
CSS
Es el lenguaje de reglas que usamos para aplicar estilos al contenido web.
JavaScript
Es un lenguaje de programación con varios usos, unos de los más importantes relacionados al diseño web es que permite crear webs dinámicas y agregar funcionalidades a estas.

Algunas cosas que se pueden hacer con JavaScript

  • Guardar valores en variables.
  • Operaciones con "strings".
  • Ejecutar código en respuesta a ciertos eventos.

Una de las aplicaciones más importantes e interesantes de JavaScript es la disponibilidad de las "APIs" (application programming interfaces), las cuales en esencia son código predefinido que permiten implementar funciones de una forma sencilla. Las "APIs" se pueden dividir en dos:

APIs del navegador
Estas son las APIs que vienen con el propio navegador. La más utilizada generalmente es DOM (Document Object Model) API.
APIs externas (third party APIs)
Como su propio nombre lo indica, estas no se encuentran de forma nativa en los navegadores. El código de estas se tiene que obtener de una fuente externa. Un ejemplo puede ser "Google maps API", la cual permite hacer implementaciones de google maps en una página.

Seguridad del navegador

Al cargar una página el navegador nos muestra todo nuestro código en conjunto (HTML, CSS, JS), es decir, el navegador toma todos nuestros archivos y nos muestra el resultado del conjunto de estos. Por seguridad, generalmente cada ventana en el navegador es un propio "visualizador", es decir que cada pestaña ejecuta el código de su pagina por separado, de esta forma una pestaña no puede modificar o realizar acciones en otra. Esto no significa que no sea posible realizar dicha acción, sin embargo, no es el comportamiento por defecto.

Orden de ejecución

El navegador al encontrar código JS, este lo ejecuta y lo lee de arriba hacia abajo (generalmente), esto significa que si por ejemplo, escribimos algo que requiere de un código que aún no está disponible, esto generaría un error.

Código interpretado vs código compilado

JavaScript es un lenguaje interpretado y en un lenguaje de este tipo, el código se ejecuta de arriba hacia abajo y el resultado es mostrado de forma inmediata, es decir, conforme se vaya procesando. El código no se tiene que transformar antes de poder ser ejecutado. Mientras que en un lenguaje compilado, el código es transformado (compilado) a otro lenguaje antes de ser ejecutado. Por ejemplo, en el caso de C++, el código primero es transformado a código máquina y este luego es ejecutado. El programa es ejecutado desde un formato binario, el cual es generado con base en el código original.

Actualmente, los interpretadores de JS más modernos usan una técnica llamada "just-in-time-compiling" para mejorar el rendimiento. El código JS es compilado en un formato binario mientras este es utilizado, de esta forma puede ser ejecutado lo más rápido posible, sin embargo, JS sigue siendo considerado un lenguaje interpretado ya que esta "compilación" ocurre en el "run-time" a diferencia de un lenguaje compilado que tiene tanto un "compile-time" (antes de ejecutarse) como un "run-time" (al estar en ejecución).

Dinámico vs estático

La palabra "dinámico" describe el cambio, es decir, la habilidad de mostrar diferente contenido en una página o aplicación. Por ejemplo, podemos crear una tabla que cambie dependiendo de información extraída de un servidor. Mientras que una web estática, a diferencia de una dinámica, no cambia, es decir que siempre muestra el mismo contenido.

Agregando JS a una página

Existen varias formas de agregar código JS en una página web.

JavaScript interno
El código se agrega entre las etiquetas <script> ... </script>, el cual debe ir justo antes de la etiqueta </body>, esto se debe a la forma en que el contenido es cargado. Haciendo esto nos aseguramos de que el código JS se ejecutará una vez cargado todo el documento HTML.
JavaScript inline
Es posible agregar código JS directamente en los elementos HTML, sin embargo, esto es una mala práctica y debería evitarse lo más posible.
JavaScript externo
Al igual que con CSS, podemos crear un archivo externo para contener nuestro código JS, esta es la práctica más común. Este se agrega con el atributo "src" en la etiqueta <script> ... </script> para indicar la ubicación del archivo.

Estrategias para el cargado de scripts

Todo el HTML en una página es cargado conforme aparece, al usar JS para manipular este, el código no funcionará si este es cargado antes que el propio HTML. Para evitar estos posibles errores, existen diferentes métodos:

  • Cuando el código JS está al final del documento HTML, este solo ejecutará después de que el documento HTML se haya cargado por completo.
  • Usando "type='module'", al ser tratado el código JS como un módulo, el navegador espera a que se cargue el contenido HTML para luego ejecutar el módulo JS.
  • Usando el atributo "defer", esto sirve para JS no modular.
  • Usando el atributo "async" cuando los scripts no necesitan que el documento HTML se cargue.

La práctica más común actualmente es utilizar JS como módulos.

Comentarios en JavaScript

Al igual que con HTML y CSS, en JavaScript también es posible incluir comentarios, es decir, texto que es ignorado al ejecutar el código JS. Los comentarios son útiles para documentar o agregar recordatorios.

  • // (comentarios de una línea)
  • /* */ (comentarios de bloque)
