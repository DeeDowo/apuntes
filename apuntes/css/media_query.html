Fundamentos de "media query"

Como mencionado anteriormente, los "media queries" son una especie de "test" que permiten aplicar reglas solo cuando este "test" es aprobado. Por ejemplo, que el tamaño de una pantalla sea de un cierto tamaño. Estas son especialmente útiles en el RWD ya que nos permiten hacer desde cambios sutiles a cambios extremos dependiendo del espacio disponible o alguna otra regla.

Estructura básica de un "media query"

Media type
Indica para qué tipo de medio son estas reglas, cuando este no se especifica, por defecto se aplica "all".
  • Contenido impreso (print)
  • Contenido mostrado en pantalla (screen)
  • Ambos (all)
Media expression
Indica una regla o "test" que se debe pasar para que las reglas declaradas sean aplicadas.
Reglas CSS
Los diferentes bloques de reglas que queramos aplicar al contenido que pase el "test" definido.

Orientación

Otra regla bastante soportada es "orientation" para la orientación, cuyos valores son:

  • portrait (vertical)
  • landscape (horizontal)

En este ejemplo el "foreground" del elemento "body" será de color azul cuando la orientación sea de tipo "landscape" (horizontal). Las páginas vistas en una computadora de escritorio o laptop, por defecto tienen la orientación de tipo "landscape".

Punteros

La regla "hover" verifica si el usuario está usando algún puntero. La navegación por teclado o en una pantalla táctil no cuentan como "hover".

La regla "pointer" identifica el tipo de medio para apuntar que se está utilizando, puede tomar tres valores diferentes:

Fine
El valor "fine" indica que el usuario está utilizando algo como un mouse o track pad, esto es útil ya que podemos crear nuestros estilos basados en la idea de que el usuario puede seleccionar elementos con precisión.
Coarse
El valor "coarse" indica que el usuario está utilizando una "touchscreen", esto es útil ya que podemos crear nuestros estilos basados en la idea de que el usuario no puede seleccionar elementos con tanta precisión.
None
El valor "none" significa que el usuario no tiene ningún dispositivo para seleccionar, es decir, este está utilizando ya sea un teclado para navegar o comandos de voz.

Utilizar la regla "pointer" puede ser de ayuda para crear diferentes interfaces para diferentes medios y de esta forma dar una mejor experiencia al usuario.

Media queries de mayor complejidad

Similar a los lenguajes de programación, es posible utilizar operadores lógicos para crear "media queries" de mayor complejidad combinando múltiples "tests". Estos operadores son:

OR
El operador "," indica que las reglas se aplicarán mientras se cumpla al menos uno de los "tests" indicados.
AND
El operador "and" indica que las reglas se aplicarán cuando se cumplan ambos "tests" comparados.
NOT
El operador "not" indica que las reglas se aplicarán cuando un "test" en particular no se cumpla.

Cómo escoger los puntos de quiebre

Anteriormente se solían utilizar los diferentes tamaños de los diferentes dispositivos disponibles, sin embargo, actualmente esa no es la mejor práctica ya que ahora existe una variedad mucho mayor de estos.

El tipo de acercamiento más utilizado actualmente es modificar el diseño de algo cuando este se "rompe", es decir, cuando el diseño comienza a perder significado o el contenido ya no puede ser contenido de forma clara. Con este tipo de acercamiento realmente no importan los tamaños específicos de los diferentes dispositivos disponibles.

En las "devtools" de Firefox podremos encontrar el "responsive design mode", en este modo podemos hacer el "viewport" más grande o más pequeño para ver en qué puntos nuestro diseño necesita un cambio o algunos ajustes.

Diseño responsivo "mobile first"

Generalmente existen dos enfoques principales a la hora de diseñar una página web, estos son:

  • Diseñar desde el mayor tamaño e ir reduciendo a partir de este.
  • Diseñar desde el menor tamaño e ir aumentando a partir de este.

El segundo es conocido como "mobile first" y suele ser el enfoque más común y práctico.

El diseño para dispositivos más pequeños generalmente solo será una columna, por ello suele ser solo un diseño simple con ajustes mínimos y por ello suele ser muy similar al flujo normal por defecto.

Algo a considerar es que no siempre es necesario el uso de "media queries", algunos de los diferentes métodos para layout como "flexbox" o "grid" suelen tener comportamientos que son responsivos por defecto, solo hay que saber utilizarlos, esto es algo que se consigue entendiendo el comportamiento de las propiedades que estos nos proveen y con la práctica.
