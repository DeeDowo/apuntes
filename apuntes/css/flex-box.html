Flexbox

Flexbox es un modelo de layout en CSS diseñado para distribuir y alinear elementos de manera eficiente en una sola dimensión principalmente (ya sea horizontal o vertical). Es ideal cuando queremos que el contenedor se adapte de forma dinámica a sus elementos hijos, en lugar de forzar un diseño rígido.

Para empezar, flexbox depende de 2 factores principalmente, el contenedor de los elementos flexbox y los elementos flexbox, es decir, primero hay que definir qué elemento tendrá la propiedad "display:flex" esto lo convierte en un contenedor (flex container), esto lo convertirá en un elemento block, para evitar esto se puede usar la propiedad "display:inline flex". Una vez convertido en un contenedor a un elemento, todos sus hijos ahora serán elemento flex (flex items).

Por defecto, al asignar a un elemento el valor "flex", los elementos dentro de este se acomodarán en forma de una sola fila, con el mismo tamaño y altura.

flex container
Es el contenedor de todos los flex items
flex items
Los elementos anidados dentro del flex container (solo los hijos directos)

El modelo flex

Cuando los elementos son distribuidos como flex items, estos se distribuyen en dos ejes.

Main axis
Es el eje principal, es decir, la dirección en que los "flex items" son mostrados (fila o columna). El inicio y fin de este eje son llamados "main start" y "main end", la distancia entre estos puntos es llamado "main size".
Cross axis
Es el eje perpendicular al "main axis", en otras palabras, es el otro eje respecto al principal. Similar al "main axis", posee un "cross start", "cross end" y "cross size".

Columnas o filas

La propiedad "flex-direction", permite indicar cuál será el "main axis" ("row" o "column"), el valor por defecto es "row".

"Wrapping"

Los "flex items" dentro de un contenedor crearán un desbordamiento cuando el tamaño del elemento padre sea superado, esto sucede ya que por defecto los navegadores tratarán de acomodar a los hijos siempre en una sola fila o columna, para evitar esto existe la propiedad "flex-wrap:wrap", la cual permite que automáticamente cuando se detecte que los flex items superarán el tamaño del contenedor, estos se posicionen en una nueva línea ya sea horizontal o vertical dependiendo del "flex-direction" establecido.

Shorthand flex-flow

Al igual que con muchas otras propiedades, flexbox también tiene un shorthand para "flex-direction" y "flex-wrap", este siendo "flex-flow". De esta forma podemos tanto indicar la dirección de nuestro flujo como el tipo de "envoltura".

Tamaño variable con "flex"

La propiedad "flex", permite repartir el espacio restante de forma fraccionaria, es decir, si tenemos un contenedor cuyo espacio no se cubre en su totalidad, este espacio restante puede ser repartido usando flex. La forma en que "flex" funciona es dividiendo todo el espacio restante en una cantidad de porciones y luego repartiendo estas.

En un ejemplo donde hay tres "flex items" y los primeros dos tienen la propiedad "flex:1;" mientras que el tercero tiene la propiedad "flex: 2;", el espacio restante se dividirá en 4 partes y estas serán repartidas dependiendo de la propiedad flex de cada elemento.

  • flex: 1; (Una fracción del espacio a repartir)
  • flex: 2; (Dos fracciones del espacio a repartir)

La propiedad "flex" también permite indicar el factor de reducción y un tamaño base mínimo. El factor de reducción se aplica cuando un "flex container" no puede contener en su totalidad a los "flex items", este factor dicta en qué porción se reducirán los elementos para evitar un overflow. El tamaño mínimo quiere decir que si un "flex item" tiene un valor menor al indicado, este crecerá, pero si el tamaño del "flex item" es mayor, este ignorará el tamaño base. Luego el resto de espacio se repartirá en partes iguales a menos que se indique lo contrario. Por ejemplo "flex: 1 0 100px" indica que el espacio sobrante se repartirá equitativamente, que no se reducirá y que el valor inicial de cada "flex item" será de 100px.

Flex shorthand vs longhand

La propiedad "flex" es un shorthand para 3 propiedades:

flex-grow
Indica qué porción del espacio sobrante se repartirá para cada elemento.
flex-shrink
Indica en qué porción se reducirá en tamaño un elemento cuando haya un "overflow".
flex-basis
Indica el tamaño base para cada "flex item", si estos tiene un valor superior, ignoran flex-basis.

Alineación horizontal y vertical

Los "flex items" se pueden acomodar de forma respectiva a su contenedor, es decir, podemos alinear "flex items" sobre el "main axis", "cross axis" o sobre ambos.

Eje Propiedad Valor Descripción
Main axis (eje principal) justify-content start Los elementos se acomodan desde el inicio del "main axis".
end Los elementos se acomodan al final del "main axis".
center Los elementos se acomodan al centro del "main axis".
space-around Los elementos se distribuyen dejando espacios iguales entre ellos, incluso al inicio y fin.
space-between Similar a "space-around", los elementos se acomodan dejando espacios iguales entre ellos, con la diferencia de que no se dejan espacio al inicio y fin.
Cross axis (eje secundario) align-items stretch Los elementos se estiran para llenar todo el espacio en el "cross axis", cuando el contenedor no tiene una dimensión fija para el "cross axis", los "flex items" tendrán la altura del "flex item" más alto entre ellos.
center Los elementos mantienen sus dimensiones intrínsecas, pero se centran a lo largo del "cross axis".
start Los elementos se acomodan al inicio (parte superior normalmente) del "cross axis".
end Los elementos se acomodan al final (parte inferior normalmente) del "cross axis"

La propiedad "align-items" (cross axis) se puede sobrescribir para cada "flex item" por separado utilizando la propiedad "align-self".

Orden de los flex items

Los "flex items" pueden moverse sin la necesidad de afectar al documento mismo, para esto se usa la propiedad "order", que por defecto todos tiene un valor de "0", eso significa que los elementos se mostrarán conforme aparezcan en el documento, un valor más alto hará que aparezca después mientras que un valor más bajo hará que aparezcan antes. Sin embargo, esto no afecta el orden real de los elementos de la página, por lo mismo es mejor evitar su uso y solo basarse en el propio documento para cambiar el orden de los diferentes elementos.

Algunas aplicaciones comunes de flexbox:

  • Centrar contenido de manera vertical y horizontal
  • Distribuir espacio disponible entre elementos de manera proporcional
  • Ordenar o reorganizar elementos sin modificar el HTML

Otra práctica común es anidar cajas "flexbox", es decir, podemos hacer que un "flex item" sea a la vez un "flex container". En otras palabras, es posible anidar diferentes "flex layouts".
