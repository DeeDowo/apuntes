Box model

La forma más fácil de comprender CSS, es entiendo que todo en CSS es "una caja", en otras palabras, todo en CSS es un contenedor.

Block e Inline

Block ("Block boxes") e Inline ("Inline boxes") son los 2 tipos principales de "display" que indican de qué manera se comporta un elemento en la pagina y el cómo es su relación con el resto de elementos en esta. La disposición "block" e "inline" son el por defecto en que se comporta la web.

display: block;
  • La "caja" crea un salto de linea tanto antes como después
  • La "caja" tiene width y height
  • El padding y el margin empujan otras cajas
  • Si el "width" no es especificado, la caja tomará todo el ancho disponible
display: inline;
  • La "caja" no genera nuevas lineas
  • La "caja" no tiene ni "width" ni "height"
  • El padding y el margin no mueven otras cajas, solo crean espacio en la dirección de la linea

Un par de formas de identificar con facilidad el tipo de "display" que posee una "caja" es:

  • Cuando los elementos se agregan uno a lado de otro, son de "display: inline;"
  • Cuando los elementos se agregan uno debajo de otro, son de "display: block"

Inner y outer display

Cada elemento posee un "inner display type" y un "outre display type". El "inner display type" hace referencia a el cómo se distribuyen los elementos dentro de una "caja" mientras que el "outer display type" hace referencia a el cómo se comportan los elementos en relación al resto de elementos. "inline-block" es otro tipo de "display" que en esencia es una combinación de ambos tipos (block e inline), de ahí su nombre, comparte todas las características de un elemento de tipo "block", con la excepción de no forzar un salto de linea antes y después de este.

Qué es el Box-model?

Define el conjunto de partes que forman a una "caja" (margin, border, padding y contenido) y el cómo interpreta el navegador dichas "cajas".

Existe dos modelos, el modelo estándar y el modelo alternativo. por defecto, los navegadores usan el modelo estándar.

Partes de una caja

Todo elemento en CSS tiene 4 elementos:

  • width/height: Depende del "Box-mode".
  • padding: Espacio blanco al rededor del contenido.
  • margin: Espacio blanco entra elementos.
  • border: División entre el "margin" y el elemento (envuelve el contenido).

Modelo estándar

En el modelo estándar, llamado "content-box", el tamaño de la "caja" es la sumatoria de los elementos, es decir:

  • ancho/largo: width + 2 x padding + 2 x border
  • alto: height + 2 x padding + 2 x border

Para el modelo estándar, el "width/height" son solo las medidas del contenido interno de una "caja" (sin contar padding ni border).

Modelo alternativo (recomendado)

La principal diferencia con el modelo estándar es que el "width/height" son el total de "caja" y no solo una parte de esta, es decir, en el modelo alternativo, llamado "border-box", el "width/height" representan la dimensiones totales que vemos de la "caja".

Más sobre margin, padding y border

Visto en el punto anterior, estos son los elementos de una "caja" de CSS aparte del contenido de esta.

Elemento Descripción
margin El espacio invisible al rededor de las "cajas", puede empujar a otros elementos. Un margen puede tener tanto valores positivos o negativos. "margin" es una propiedad "shorthand", esta manipula los 4 lados de la "caja" a la vez y puede ser divida en propiedades más especificas (margin-top, margin-bottom, margin-left, margin-right).
padding Es el espacio que se encuentra entre el contenido de la "caja" y el "border", empuja al contenido desde el borde hacia adentro. A diferencias de los margenes, no se pueden tener paddings negativos. Sin embargo, al igual que con "margin", "padding" es una propiedad "shorthand" que manipula los 4 lados a la vez y puede ser divida en propiedades más especificas (padding-top, padding-bottom, padding-left, padding-right).
border Es la división que separa el contenido y "padding" del "margin", puede tener una variedad de tamaños, colores y estilos. Al igual que con "margin" y "padding", la propiedad "border" es un "shorthand" que manipula los 4 lados a la vez y puede ser divida en propiedades más especificas (border-top, border-bottom, border-left, border-right).

Colapso de margenes

Cuando los margenes verticales de 2 elementos diferentes "chocan", pueden provocar diferentes comportamientos, dependiendo de sus valores.

Caso Resultado
Dos margenes positivos Cuando Dos margenes positivos "chocan", aquel que tenga el mayor valor será el que se respetará, en otras palabras, el margen más grande será el margen entre estos Dos elementos.
Dos margenes negativos Cuando Dos margenes negativos "chocan", se respetará aquel que tenga el menor valor, es decir, aquel más alejado del 0.
Un margen positivo y Un negativo Cuando un margen positivo "choca" con uno negativo, estos se suman (margen1 + (- margen2) = margen1 - margen2).
