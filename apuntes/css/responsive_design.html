Responsive design

RWD (responsive web design) es un acercamiento al diseño web, principalmente hace referencia a crear diseños adaptables a toda clase de entornos.

Media query

El uso de "media queries" es una práctica común para estilar una página web adaptable a diferentes resoluciones o circunstancias. Los puntos donde los "media queries" se activan son conocidos como puntos de quiebre o "break points".

Uno de los acercamientos más comunes al diseñar una página es conocido como "mobile first". Este método hace referencia a estilar una página desde un tamaño más compacto e ir progresivamente adaptando este diseño a distintos tamaños.

Al usar puntos de quiebre en una página web, la mejor práctica es que estos estén definidos con unidades relativas en lugar de unidades fijas. Las reglas dentro de los "media queries" pueden ir desde algunas propiedades hasta enlazar un archivo CSS completamente aparte.

Si bien los "media queries" sirven para el RWD, estos no son ni requeridos ni necesarios para aplicar este tipo de enfoque de diseño. Existe una gran variedad de elementos HTML y propiedades CSS que por defecto ya son "responsive".

Responsive layout

Principalmente "Flexbox" y "Grid" son "responsive" por defecto.

Es por ello que "flex" se suele usar especialmente para elementos que pueden funcionar de forma "individual" mientras que "grid" se suele utilizar especialmente para un conjunto de elementos en donde el objetivo principal es mostrar este contenido con un cierto "formato".

Cosas a considerar al usar media

  • Utilizar un formato correcto al igual que optimizar los diferentes archivos.
  • Existen alternativas en CSS para agregar efectos o formas sin la necesidad del uso de archivos externos.
  • Es posible usar "media queries" para tener diferentes archivos para diferentes dispositivos.

Usando "viewport units"

El usar estas unidades de forma individual ocasionará que sin importar la cantidad de zoom-in o zoom-out realizado el texto no cambiará de tamaño ya que este está ligado al tamaño de la ventana. Sin embargo, esto puede ser usado a nuestro favor. Si usamos la función "calc()" y dentro de ella realizamos la suma de un valor con una unidad relativa y un valor con una unidad relativa al viewport de la página, este tamaño sí variaría dependiendo del zoom-in, zoom-out y el tamaño por defecto del navegador. El resultado de esta operación sería un valor fijo "adaptable", de esta forma podemos solo especificar una vez el tamaño de nuestras fuentes.

La etiqueta meta "viewport"

Esta etiqueta hace que los navegadores móviles usen el "width" real del dispositivo como el "viewport width". Esto se debe a que algunos navegadores móviles tienden a "mentir" sobre este aspecto, es decir, en lugar de usar el tamaño real del dispositivo, utilizan otra medida arbitraria, generalmente "980px".
