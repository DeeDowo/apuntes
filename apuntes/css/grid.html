Grid

"grid" es otro método para distribuir los elementos en una página web que permite organizar el contenido en filas y columnas, o más bien, permite organizar el contenido de una página en una retícula. "grid" permite crear distribuciones más complejas y con mayor facilidad que el resto de métodos presentados. Generalmente "grid" se usa como un esqueleto-base para el resto de técnicas.

Grid layout

Es un conjunto de líneas horizontales y verticales que crean sectores en los cuales podemos posicionar nuestros elementos. Los elementos generales de un "grid" son columnas, filas y espacios entre estas o "gutters" (espacios entre columnas y filas).

Para definir un "grid", primero tenemos que tener un elemento al cual le queremos dar esta propiedad y que este elemento tenga elementos hijos a los cuales queramos distribuir.

Al igual que con "flex", tenemos que fijar la propiedad "display: grid" al contenedor en cuestión. Por defecto "grid" agrega una sola columna en donde se van acomodando los elementos en diferentes filas, similar al comportamiento por defecto.

Para definir columnas diferentes podemos aplicar la propiedad "grid-template-columns", la cual define la cantidad de columnas que se crearán. Por ejemplo, "grid-template-columns: 200px 200px 200px;" creará un "grid" de 3 columnas donde cada una tendrá un tamaño de 200px.

Sin embargo, no es recomendable aplicar tamaños fijos a elementos en general ya que abren la posibilidad a "overflows" de contenido. "grid" nos permite el uso de "fr", la cual es una unidad fraccionaria, es decir, una porción que distribuye el espacio en partes iguales. Por ejemplo, si usamos "grid-template-columns: 1fr 1fr 1fr;" esto nos creará 3 columnas, similar al punto anterior, pero con la diferencia de que estas 3 columnas se dividirán equitativamente el espacio disponible. El proceso para esto es de la siguiente forma:

  1. Primero se hace una división del espacio total disponible entre la cantidad total de "fr" que hayamos especificado.
  2. Luego estas se dividen dependiendo de la porción asignada a cada columna.

Por ejemplo, si tenemos "grid-template-columns: 2fr 1fr 1fr;", esto hará que el espacio total se divida en 4 partes y que a la primera columna se le asignen 2 de estas 4 partes, mientras que a la segunda y tercera columna se le asignarán solo una de estas partes.

Las unidades fijas se pueden combinar con las unidades fraccionarias. En este caso, primero se asignan las fijas y luego con el espacio restante disponible se calculan las fraccionarias.

Espacios en la retícula

Para poder crear espacios entre filas, columnas o ambas, tenemos 3 propiedades a nuestra disposición:

  • column-gap (columnas)
  • row-gap (filas)
  • gap (shorthand para "column-gap" y "row-gap")

Los espacios pueden ser de cualquier tipo de unidad, ya sea relativa o absoluta, pero no se pueden usar "fr".

Grids explícitos e implícitos

Grid explícito:
Es aquel que definimos. Creado usando "grid-template-columns" o "grid-template-rows".
Grid implícito:
Genera columnas o filas de manera automática al notar que no hay suficientes espacios para los elementos.

Por defecto el tamaño de las columnas o filas creadas implícitamente tienen un tamaño "auto", es decir, son lo suficientemente grandes para contener su contenido. Si lo que queremos es asignar un tamaño fijo a estos, podemos utilizar las propiedades "grid-auto-columns" o "grid-auto-rows" dependiendo del caso y con estas asignar el tamaño que tendrán estas cuando se generen.

La función minmax() en grid

El fijar tamaños a cualquier elemento no suele ser la mejor opción y en grid no es la excepción. Fijar tamaños abre la ventana a posibles "overflows". Para evitar esto podemos usar la función "minmax()", la cual nos da más control para estos casos. Nos permite fijar un tamaño mínimo y dejar en "auto" cuando el contenido de un elemento sea superior a este. Por ejemplo, teniendo "minmax(100px, auto)", estamos diciendo que el tamaño mínimo de estos contenedores será de 100px, pero en el caso de que este tamaño se llegara a sobrepasar, el contenedor crecerá de forma automática. Algo a considerar, es que esta expansión solo afecta a la fila o columna en cuestión.

Tantas columnas como sea posible

Podemos combinar todo lo mencionado anteriormente para crear un "patrón" que creará columnas de forma automática, utilizando todo el espacio disponible. Esto se hace de la siguiente forma: 

display: grid;
grid-template-columns: repeat(auto-fit, minmax(100px, 1fr));
grid-auto-rows: minmax(50px, auto);
gap: 20px;

Este código realiza lo siguiente:

  1. Asignamos "display: grid;".
  2. Asignamos "grid-template-columns: repeat(auto-fit, minmax(100px, 1fr))", utilizando la función "repeat", luego usamos la palabra clave "auto-fit", que significa que se crearán tantas columnas como sea posible, y por último la función minmax(), la cual indica qué tamaño mínimo tendrá cada columna y que de ser posible un tamaño superior, cada una será de "1fr", es decir, del mismo tamaño. En esencia esta declaración está asegurando que siempre tendremos el espacio lleno de columnas de al menos 100px, que todas serán del mismo tamaño y que estas ocuparán todo el espacio disponible.
  3. Asignamos "grid-auto-rows: minmax(50px, auto);" eso significa que cada fila creada de forma automática tendrá al menos 50px de altura, pero que si necesita más, se expandirá para abarcarlo.
  4. Por último asignamos "gap: 20px;", con esto decimos que entre cada columna y fila habrán 20px de espacio como separación.

Ubicación basada en "líneas"

Una vez entendidos los conceptos fundamentales para crear un "grid", podemos comenzar a usarlo, es decir, comenzar a acomodar contenido dentro de la retícula creada.

Lo primero para poder posicionar nuestro contenido utilizando "grid", es entender el cómo se "ve" este. Para el navegador solo son líneas que separan sectores, cada una tiene un número comenzando desde el "1". Por ejemplo, si tenemos un grid con 2 columnas y 2 filas, eso significaría que tenemos 4 cuadrantes disponibles:

  • "1-2/1-2" (columna/fila)
  • "2-3/1-2" (columna/fila)
  • "1-2/2-3" (columna/fila)
  • "2-3/2-3" (columna/fila)

Usamos estas líneas como referencia para poder posicionar nuestros elementos dentro de un "grid". La dirección de estos "números" depende del "writing mode" del navegador (depende de idioma).

Volviendo al cómo ubicar nuestros elementos, para esto tenemos varias propiedades que indican una línea de inicio y una línea de fin.

grid-column-start
Para indicar la columna desde dónde inicia nuestro elemento.
grid-columns-end
Para indicar la columna en dónde finaliza nuestro elemento.
grid-row-start
Para indicar la fila desde dónde inicia nuestro elemento.
grid-row-end
Para indicar la fila en dónde termina nuestro elemento.
grid-column
Permite indicar tanto el inicio como el final de las columnas separadas por un '/'.
grid-row
Permite indicar tanto el inicio como el final de las filas separadas por un '/'.

Cuando aplicamos "display: grid" a un elemento, lo convertimos en un contenedor "grid" y todos sus elementos se acomodarán en el sector que especifiquemos. Cuando no especificamos explícitamente qué lugar o cuántos lugares tomará cada elemento, estos se acomodan conforme aparezcan en el documento HTML.

Ubicación basada en "areas"

Una forma alternativa para poder posicionar elementos en un "grid", más sencilla e intuitiva, es a través de la propiedad "grid-template-areas" donde a cada sector del "grid" se le asigna un nombre y luego estos nombres al ser referenciados con la propiedad "grid-area" hará que dicho elemento se posicione en el sector o sectores indicados.

La recomendación principal para su uso es saber de antemano la clase de distribución que queramos hacer y al asignar los nombres de los cuadrantes, utilizar los propios nombres de los elementos o algunos autodescriptivos. Un ejemplo de la notación del "grid-template-areas" es:

  • Todos los sectores deben ser nombrados.
  • Para usar más de un sector se debe repetir el nombre.
  • Para dejar un sector vacío se usa un '.'.
  • Las áreas deben ser formas rectangulares, no pueden haber formas 'L' o de cualquier tipo diferente al rectangular.
  • Las áreas iguales solo pueden ser contiguas, no pueden estar dispersas en cualquier parte.

El uso de "grid-template-areas" es muy útil para un layout general, ya que es muy fácil de expresar y entender a simple vista.

Grids anidados y "subgrid"

Al igual que con flex, es posible anidar "grids" dentro de "grids", esto crea un "grid" anidado.

Un valor que se puede usar en "grids" anidados es el de "subgrid" el cual sirve para que estos "grids" anidados sigan las líneas del "grid" padre, es decir, las filas y columnas de este. Por ejemplo, si un "grid" anidado ocupa 2 columnas de un "grid" padre de 3 columnas y en este "grid" anidado usamos "grid-template-columns: subgrid;" este "grid" anidado tendrá las líneas de las 2 columnas que ocupa en el padre. En este ejemplo solo declaramos "subgrid" para las columnas, lo que significa que las filas se crearán de forma automática o las podemos definir nosotros, pero estas no estarán ligadas al "grid" del padre. Sin embargo, igual podríamos aplicar "subgrid" para estas.
