Más sobre selectores

Posiblemente la parte más importante en CSS, los selectores son los indicadores de a qué se le aplicarán las reglas declaradas. Estos pueden combinarse de múltiples formas, complejas o sencillas. En otras palabras, los selectores indican el "camino" hacia el elemento al cual queremos aplicar ciertos estilos. Existen muchas formas de definir un selector, algunos ejemplos:

  • Elementos HTML
  • Clases en elementos HTML
  • ID's en elementos HTML

Selector universal

Un selector de particular importancia es el selector universal '*', selecciona todo el documento. Un uso "común" es en conjunto con los combinadores o para "normalizar" estilos a gran escala en el documento.

  • * {...} (todos los selectores de un documento)
  • article * {...} (Todos los "hijos" dentro de un elemento)
  • article > * {...} (Todos los "hijos directos" dentro de un elemento)
  • article ~ * {...} (Todos los "hermanos" posteriores de un elemento)
  • article + * {...} (Todos los "primeros hermanos" de un elemento)

Elementos HTML como selectores

Todos los elementos HTML pueden ser utilizados como selectores. El principal uso de esto es para poder aplicar reglas generales a múltiples elementos de una forma sencilla y práctica.

  • h1 {...} (Se aplican estilos a todos los elementos "h1")
  • p {...} (Se aplican estilos a todos los elementos "p")
  • section {...} (Se aplican estilos a todos los elementos "section")
  • body {...} (Se aplican estilos a todo el elemento "body")
  • span {...} (Se aplican estilos a todos los elementos "span")

Algo a considerar es que estos selectores, al ser tan generales, no deberían usarse justamente para reglas específicas.

Clases HTML como selectores

El usar clases como selectores es la práctica más común al usar CSS. Permite aplicar reglas solo a elementos que posean dichas clases. Estas son sensibles a mayúsculas y minúsculas. A diferencia de los selectores de elementos generales, estas permiten aplicar reglas a elementos más específicos. Es posible usar múltiples clases a la vez para selectores aún más específicos.

  • .clase {...}
  • .clase.clase2 {...}
  • .clase.clase2.clase3 {...}

Elementos con clase

Selecciona todos los elementos que a su vez posean una clase en particular. Una práctica común es combinar diferentes tipos de selectores, por ejemplo, elementos generales con clases, esta práctica reduce el "scope" (alcance) de las reglas.

  • h1.clase {...}
  • p.clase.clase {...}
  • span.clase {...}

Selectores por ID

Al igual que con las clases, estos selectores son sensibles a mayúsculas y minúsculas, incluyen el prefijo '#' seguido del "ID". Selecciona solo un elemento (un ID debe ser único), aquel con el ID indicado.

  • #ID {...}

En CSS, los selectores por ID son aquellos que tienen la mayor especificidad posible.

Lista de selectores

Cuando tenemos reglas que se pueden aplicar a más de un selector, lo mejor es agruparlos para no tener que reescribir reglas múltiples veces. En esta "lista de selectores", cada elemento se separa por una coma (',').

  • h1, #ID, .clase, .clase1.clase2, p > span {...}

Algo a considerar al usar "listas de selectores" es que si alguno de los selectores tiene un error de sintaxis ("..clase", "# ID", etc.), las reglas no se aplicarán a ningún elemento.

Selectores de atributos

Una práctica no tan común, pero también es posible seleccionar elementos que posean ciertos atributos asignados en el documento HTML. Estos se pueden clasificar de la siguiente forma:

Presencia
Selecciona elementos que tengan un cierto atributo:
  • [atributo] {...}
Valor de atributos
Selecciona elementos que tengan un cierto valor en un atributo:
  • [atributo="valor_exacto"] {...}
  • [atributo~="contiene el valor exacto solo o como parte de una lista de clases"] {...}
  • [atributo|="contiene el valor exacto o como inicio + '-' (importa la ubicación)] {...}
Coincidencia parcial del valor de un atributo
Selecciona elementos que tengan una parte de un string en su valor
  • [atributo^="inicia con un texto"] {...}
  • [atributo$="termina con un texto"] {...}
  • [atributo*="contiene un texto en cualquier punto"] {...}

Selectores de pseudoclases y pseudoelementos

Otra práctica común es el uso de pseudoselectores y pseudoclases. La principal ventaja de estos es que se "adaptan" a contenido variable.

Pseudoclases
Seleccionan un elemento que está en un estado específico. Existen muchas y sirven para propósitos específicos, como el primer elemento de un tipo o cuando se pasa el cursor sobre un elemento.
Usualmente actúan como si los elementos tuvieran cierta clase aplicada sin tener que aplicarla explícitamente en el documento HTML, de ahí su nombre, lo cual crea un código más flexible. Las pseudoclases comienzan con ':' seguidas de su nombre clave.
No es necesario que una pseudoclase esté en conjunto con un selector. De esta forma puede tener un comportamiento más "abierto", sin embargo, no es muy recomendable.
  • p:first-child {...} (Selecciona un párrafo cuando es el primer hijo de algún elemento)
  • article > p:first-child {...} (Selecciona un párrafo cuando es el primer hijo de un article)
  • article ul li:first-child {...} (Selecciona el primer hijo "li" anidado en una "ul" anidada en un article)
  • :hover (Cuando el cursor se pasa sobre un elemento)
  • :nth-child() (Selecciona a un hijo de un tipo de elemento, toma como parámetro el número del hijo)
  • :last-child (Selecciona un elemento cuando es el último hijo)
  • :active (Cuando se hace "click" en un elemento)
Pseudoelementos
Los pseudoelementos tienen un comportamiento similar a las pseudoclases. Sin embargo, como su nombre sugiere, estos actúan como si se hubiera agregado un elemento totalmente nuevo al documento HTML.
En otras palabras, una pseudoclase simula una "señalización", mientras que un pseudoelemento simula un elemento totalmente nuevo.
A diferencia de las pseudoclases, su notación es "::" seguida del nombre clave del pseudoelemento. Algunos pseudoelementos utilizan solo ':', eso es más por antigüedad. Actualmente todos los pseudoelementos deberían tener la notación de "::".
Si bien este comportamiento no queda claro a simple vista, lo que está ocurriendo es que "envolvemos" la primera línea de un párrafo en un "span virtual", de esta forma, sin importar si el texto aumenta de tamaño o se reduce, siempre estaremos envolviendo solo la primera línea.
  • p::first-line {...}
::before y ::after
::before (antes) y ::after (después) son dos pseudoelementos especiales y por ello merecen su propia explicación. Estos se suelen usar con la propiedad "content" para insertar contenido en el documento mediante CSS. Se suelen usar para elementos visuales como íconos por mencionar alguno.
Usamos "content" para agregar strings vacíos a los cuales luego podemos modificar para que tomen cierta forma.
Usar ::before o ::after en conjunto con la propiedad content se llama "generated content" en CSS y su uso debería estar limitado a elementos visuales. Todo contenido significativo para una página debe ser agregado por medio del documento HTML.

Combinando pseudoclases y pseudoelementos

Otra práctica común para aplicar reglas más "flexibles" es combinando las pseudoclases con los pseudoelementos. Por ejemplo, si solo quisiéramos aplicar una modificación a la primera línea del primer párrafo cuando este sea el primer hijo en cualquier article, podríamos usar el siguiente selector:

  • article p:first-child::first-line {...}

Más sobre los combinadores

Mencionado en un punto anterior, los combinadores se usan para crear relaciones entre selectores de tal forma que nos permiten seleccionar elementos basados en su localización.

Tipo Función Ejemplo
Selector de descendientes (" ") Para modificar solo elementos anidados en otro elemento (no importa el nivel de anidación siempre y cuando este exista). Si queremos afectar a un elemento <span> ... </span> solo cuando este esté anidado en un elemento párrafo.
Selector de hijos directos (>) Similar al selector de descendientes, pero este solo apunta a un primer hijo directo. Si solo queremos afectar a un elemento <span> solo cuando este sea un hijo directo de un párrafo.
Selector de hermanos adyacentes (+) A diferencia del selector de descendientes o del de hijos directos, este apunta a elementos contiguos o "hermanos". Si queremos seleccionar todos los primeros elementos párrafo que vayan después de un elemento "h2" sin elementos intermedios.
Selector de hermanos generales (~) Similar al de hermanos adyacentes, pero apunta a todos los "hermanos" en el mismo nivel después de un elemento. Si queremos seleccionar a todos los elementos párrafo después de un elemento "h2" con posibles elementos intermedios.

Es posible combinar múltiples selectores y combinadores creando cadenas más complejas.
