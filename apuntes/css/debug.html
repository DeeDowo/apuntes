Debugging CSS

Al escribir código CSS, en la gran mayoría de los casos nos encontraremos en situaciones donde haya comportamientos inesperados o situaciones donde queramos ver el funcionamiento de algún elemento de una página web externa. Para esto, el propio navegador nos provee de herramientas. Cada navegador tiene sus propias herramientas y su propia forma de acceso a estas, la forma más común es usando "F12".

Código fuente vs código mostrado

El código fuente hace referencia al documento HTML original, mientras que el código mostrado es aquel ya procesado y es el que el navegador nos muestra. En otras palabras, se hace referencia a un antes y después.

Inspeccionando el CSS aplicado

Al inspeccionar el HTML de una página web, podemos navegar a través de ella para ver las diferentes propiedades aplicadas a cada elemento, al igual que las propiedades heredadas de cada elemento. Para mayor facilidad también podemos inspeccionar directamente un elemento sin tener que navegar en el HTML, haciendo "clic" derecho sobre un elemento en particular y seleccionando "inspeccionar". Algunas acciones comunes que podemos realizar o comportamientos que podemos observar al inspeccionar algún elemento son:

  • Si este elemento usa propiedades shorthand, el navegador nos mostrará todas las propiedades asignadas a este shorthand, incluso cuando no estén en uso.
  • Podemos hacer que alguna propiedad se active o desactive con tan solo seleccionar o deseleccionar el "checkbox" que aparece al inicio de cada propiedad.
  • Podemos modificar cada una de estas propiedades y ver un cambio en vivo de estos cambios sin la necesidad de actualizar la página.
  • Podemos agregar más propiedades y visualizar sus efectos en tiempo real.
  • Podemos visualizar el "box-model" del elemento seleccionado al igual que sus medidas.

Resolviendo problemas específicos

Al trabajar con CSS pueden surgir problemas donde ciertas propiedades no se aplican de la forma que queremos o al tener que modificar CSS de sitios preexistentes, por mencionar un par de ejemplos. Esto generalmente se debe a que dichas propiedades hacen "conflicto" con propiedades con un mayor "peso". Una solución para estos problemas en particular son las "devtools", estas nos mostrarán las propiedades que están siendo sobrescritas con un formato "tachado" y a su vez nos mostrarán cuál es y de dónde proviene la propiedad que la reescribe.

Algo que siempre se debe tener en consideración es si el navegador soporta las propiedades que queremos aplicar en primer lugar, por eso es importante siempre tener claro qué tan soportadas son las propiedades que queremos usar al igual que incluir opciones para cuando ciertas propiedades no sean soportadas (fallbacks).

Una metodología para resolver comportamientos inesperados es recrear estos conflictos en entornos más controlados, es decir, tratar de extraer estos errores del documento en sí, al minimizarlos podremos encontrar con mayor facilidad el problema.
