Contenido desbordado

El desbordamiento u "overflow" es lo que sucede cuando el contenido supera el tamaño de la "caja" que lo contiene, existen diferentes métodos y propiedades para controlar este comportamiento.

CSS y la información

Cuando sucede un desbordamiento, CSS sobrepone el mostrar el contenido a ocultarlo, es decir, CSS por defecto siempre mostrará el contenido desbordado sin importar cómo afecte esto al resto de la pagina en si.

Cuando se fija el tamaño de una "caja", se tiene que tener en consideración todos estos posibles escenarios al igual que mecanismos implementados para poder manejarlos.

La propiedad "overflow"

"overflow" es unna propiedad que permite modificar el comportamiento de una "caja" cuando ocurre un desbordamiento, por defecto tiene el valor "visible", es decir, el contenido se mostrará y será visible el cómo se desborda. Algunos otros valores que puede tomar son:

hidden
Al usar este valor, forzamos al navegador a ocultar el contenido cuando ocurre un desborde, sin embargo esta no debería ser una solución, ya que ocurre una perdida de información.
scroll
A diferencia de "hidden", "scroll" es una solución más razonable ya que permite el poder desplazarse dentro de la propia "caja", de esta forma el contenido "desbordado" no saldrá de esta ni se ocultará del usuario, ya que tomará el espacio necesario mientras que la "caja" se comportará más como una "ventana" a este contenido.
Por defecto, los scrolls siempre aparecen en ambos ejes e incluso si no hay un desbordamiento, para esto tenemos varias opciones, la primera es usar overflow-x u overflow-y para solo mostrar un scroll en uno de los ejes, sin embargo, esto igual siempre los mostrara incluso si no son necesarios.
auto
El valor "auto" permite que el navegador se encargue de decidir si mostrar o no los scrolls cuando no son necesarios.

Desbordamiento no deseado

Al diseñar una web, siempre se tiene que tener en consideración qué puede provocar y cómo poder solucionar los desbordamientos, así como tratar de evitar que si quiera ocurran en primer lugar.

Es una buena practica el probar que nuestro código CSS sea "robusto", es decir, que no ocurran compartimientos inesperados.
