Introducción a CSS

Cascading Style Sheets (CSS) es el lenguaje para modificar los estilos de los elementos en un documento.

Estilos por defecto

Existen para asegurarse de que el contenido web sea entendible visualmente incluso si no los modificamos nosotros mismos. Por defecto se usa una "hoja de estilos" interna predefinida por los navegadores, esta es la responsable de cosas como:

  • Resaltar títulos por sobre el texto normal
  • Resaltar "links" por sobre el texto normal
  • Hacer que las "listas" tengan un cierto formato

¿Para qué es CSS?

Usando CSS podemos controlar con precisión cómo queremos que cada elemento de nuestra página se vea. Algunos ejemplos serían:

  • Modificando textos
  • Modificando la distribución de los elementos (layouts)
  • Efectos especiales (animaciones)

Sin embargo, siempre hay que considerar que aunque tengamos el poder de modificar cualquier elemento de cualquier forma que deseemos, este no siempre es el mejor acercamiento. Un ejemplo, las características distintivas de un elemento, como el resaltado visual de un título.

Sintaxis básica de CSS

CSS es un lenguaje basado en reglas que se pueden aplicar a uno o más elementos HTML. Donde las partes principales de estas reglas se pueden definir de la siguiente manera:

  • selector: como su nombre indica, selecciona un elemento en el documento.
  • propiedad: indica qué aspecto queremos modificar o agregar.
  • valor: como su nombre indica, se refiere al valor de una propiedad.

En el ejemplo anterior tenemos:

  • h1: Selector.
  • llaves ({}): Contenedor de las declaraciones.
  • color, font-size: Propiedades que queremos modificar.
  • dos puntos (:): Asignador.
  • red, 2.5rem: Valores a asignar.
  • punto y coma (;): Final de una declaración.

¿Cómo se aplica CSS?

Existen tres diferentes formas de aplicar CSS a un documento HTML

  • External style sheets (se enlaza con el elemento link)
  • Internal stylesheets (entre las etiquetas <style> </style>)
  • Inline styles (como atributos en elementos, no recomendable)
External stylesheets
Las instrucciones se encuentran en otro documento ".css", es la forma más común. Es posible enlazar un solo archivo ".css" a múltiples documentos para así formatear todos de la misma forma y tener un estilo consistente.
Para enlazar un documento ".css" a un documento ".html" se usa la etiqueta "link" en el elemento <head> </head>, con los atributos "href" para la URL y "rel" para indicar el tipo de relación, en este caso, "stylesheet".
Internal stylesheets
Las reglas están contenidas en el propio documento HTML. Anidadas en el elemento <style> </style>.
Es útil cuando no tenemos acceso al documento ".css" o para hacer una previsualización rápida. Sin embargo, no es útil para la reutilización ni para la práctica a gran escala.
Inline stylesheets
Utiliza el atributo "style" dentro de los elementos HTML, es donde se pueden definir una o más propiedades con sus respectivos valores, solo afecta al elemento donde se use la propiedad. A diferencia de "internal stylesheet" donde sí existe una posible utilidad, los inline stylesheet son considerados una mala práctica en general, especialmente porque combinan estilos con semántica, lo cual no es correcto.

Selectores

Visto en los puntos anteriores, es el apartado de las reglas CSS que indican a qué elemento o elementos se le aplicarán las diferentes declaraciones dentro del bloque de reglas.

Seleccionando elementos de HTML
Podemos utilizar como selector cualquier elemento "general" de HTML.
Seleccionando por clases
Especialmente útil cuando no queremos modificar todos los elementos de un tipo en nuestro archivo. Para esto podemos utilizar las clases (class), las cuales al usarlas como selectores sirven para solo modificar aquellos elementos que posean dicha clase en particular.
Seleccionando por su ubicación
Se definen relaciones entre selectores mediante diferentes símbolos (" ", +, >, ~), estos establecen una regla específica entre un par de selectores (solo se comparan dos selectores a la vez).
  • " " (Indica un elemento anidado dentro de otro sin importar el nivel)
  • > (Indica un elemento anidado dentro de otro, solo en el primer nivel)
  • + (Indica solo el primer elemento subsecuente)
  • ~ (Indica todos los elementos subsecuentes)
Seleccionando basados en su estado
El ejemplo más claro para este tipo de selectores es con un link, que tiene dos estados, antes y después de haberlo visitado. Otro ejemplo es al hacer "hover" sobre un elemento, es decir, al pasar el cursor sobre él.

Más sintaxis de CSS

Funciones
Existen valores que pueden tomar la forma de una función. Una de las principales ventajas del uso de funciones es que permiten crear valores más "dinámicos".
@rules
Las "@rules" dan instrucciones sobre cómo se debe comportar CSS en algún dado caso.
Shorthand properties
Son propiedades que permiten ingresar más de un valor a la vez.
Comentarios en CSS
Al igual que con HTML o algún lenguaje de programación, existen los comentarios en CSS, es decir, bloques de texto cuya función puede ser documentar, organizar, recordatorios, etc. Para agregar algún comentario en CSS, se hace con la notación "/* comentario */".
Espacio blanco en CSS
Estos son espacios, tabulaciones o nuevas líneas. Al igual que en HTML, el navegador ignora los espacios extras innecesarios. Sin embargo, estos son útiles para un código más claro. No obstante, no todos los espacios en blanco son innecesarios ni irrelevantes para el navegador. En el caso de propiedades que aceptan múltiples parámetros (shorthand properties), estos son necesarios para separar un valor de otro. Por otra parte, ningún nombre de parámetro tiene espacios en blanco.
