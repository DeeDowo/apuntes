Links

Un hipervínculo, link o enlace, permite conectar un documento con otro, vincular partes específicas dentro de documentos o incluso crear aplicaciones web. Los enlaces son fundamentales en la estructura de la web.

Un link se define mediante el elemento "anchor" y el atributo href, que especifica el destino del enlace, es decir, la URL.

  • <a href="www.ejemplo.com"> Ejemplo </a>

No solo los fragmentos de texto pueden ser usados como links. Cualquier elemento que esté envuelto por el elemento "anchor" puede ser un link: bloques de texto, botones, imágenes, etc.

  • <a href="..."><span> ... </span></a>

Links a elementos de un documento

Es posible enlazar una parte específica dentro de un documento HTML mediante un link. Para hacer esto, el destino debe tener un ID. Si el elemento a enlazar está en el mismo archivo, solo se indica el "ID" con el prefijo #. Cuando el elemento no se encuentra en el mismo archivo, se debe indicar la URL y el "ID" con su prefijo #.

  • <a href="#id"> Ejemplo </a> (Enlace a un elemento)
  • <a href="/carpeta/index.html#id"> Ejemplo </a> (Enlace a un elemento fuera del documento actual)

Uniform Resource Locator

La URL indica la dirección de algo: un elemento, documento, archivo, página web, etc. Existen dos tipos: URL relativas y URL absolutas.

URL relativa

Aquella que se indica tomando un punto de referencia. Si la referencia cambia, la ruta cambia.

  • "/carpeta/carpeta2/image.png"

URL absoluta

Aquella que, sin importar desde dónde se referencie, no cambia.

  • "https://www.ejemplo.com/pagina.html"

Enlaces a recursos

Cuando un link lleva a algo que no es una página, por ejemplo, un PDF, imagen o archivo descargable, es recomendable:

  • Indicar el tipo de recurso (formato y tamaño)
  • Explicar si será una descarga o una previsualización

Para una descarga se utiliza el atributo download (booleano). Se iniciará la descarga de ser posible, y si se le asigna un valor al atributo, el archivo se descargará con ese nombre.

Para abrir algo en una pestaña nueva, se utiliza el atributo target con el valor _blank. Por defecto, todos los links se abren en la misma página.

Algunas buenas prácticas para el uso de links

  • Claridad textual (palabras cortas)
  • Claridad visual (debe entenderse a simple vista que es un link)
  • No repetir nombres en la medida de lo posible

Especialmente suele ser una buena práctica indicar explícitamente o visualmente qué se realizará, ya sea una descarga, abrir en una nueva pestaña o cualquier comportamiento posible (la forma más común es por medio de iconos).
