Tablas

Las tablas organizan información estructurada en filas y columnas, deben ser claras tanto visualmente como de forma semántica. Una buena tabla es un mezcla entre contenido y estilo.

El único objetivo de una tabla debe ser mostrar el información tabular.

Elementos principales de una tabla
Elemento Función
<table> </table> (Table data) Elemento principal que envuelve toda una tabla.
<td> </td> Agrega una nueva celda contigua. Es en dónde se inserta el contenido.
<tr> </tr> (Table row) Envuelve celdas en una fila.
<th> </th> (Table header) Por defecto todos las celdas tienen el mismo peso semántico, este elemento indica que una celda contendrá un titulo, lo cual le da un mayor valor semántico.

Al crear una tabla, cuando queremos que una celda en particular ocupe más de de un celda, tenemos dos atributos para esta tarea en particular, estos son:

  • colspan="cantidad de casillas" (Indica la cantidad de celdas, horizontalmente, que abarcará una celda)
  • rowspan="cantidad de casillas" (Indica la cantidad de celdas, verticalmente, que abarcará una celda)

Accesibilidad en tablas

Si bien con los elementos y atributos mencionados previamente ya se podría crear una tabla bastante completa en si misma, existen más elementos y tributos para hacer a una tabla mas accesible.

Caption
El elemento <caption> </caption> debe ser el primer elemento anidado en una tabla y contiene una descripción clara y concisa sobre qué tratará la tabla, como un titulo para esta.
Scope
El atributo "scope" se agrega a las celdas de tipo "header" e indica la relación entre estas con el resto de celdas, es decir, indica de qué celdas es el titulo:
  • scope="col" (Es el encabezado de toda una columna)
  • scope="row" (Es el encabezado de toda una fila)
  • scope="colgroup/rowgroup" (Mismo comportamiento que los dos puntos anteriores respectivamente, pero para encabezados que abarcan multiples columnas o filas, aquellos headers que tienen el atributo "colspan" o "rowspan")

Id y headers

Un método con funcionamiento similar al atributo "scope", pero con una mayor complejidad y precisión. Su metodología es la siguiente:

  1. a cada elemento "th" se le asigna un "ID" sin espacios
  2. a cada elemento "td" se le asigna el atributo "headers" cuyo valor será el "ID" del titulo que lo define. Si un elemento tiene más de un titulo, los diferentes "ID" se separan por medio de espacios.

Este método permite crear asociaciones mas complejas, pero más claras, con la "desventaja" de estar abierto a más posibles errores que el uso del atributo "scope".

Generalmente su uso debería ser exclusivo para cuando "scope" no es suficiente, es decir, para tablas con una complejidad alta.

Otros elementos

Los elementos "thead", "tbody" y "tfoot" ayudan a organizar de mejor manera una tabla, sin embargo, la función principal de esto es para poder aplicar CSS con mayor comodidad.

  • <thead> ... </thead> (Normalmente contiene la primera fila, generalmente títulos)
  • <tbody> ... </tbody> (Normalmente contiene el resto de elementos no asignados por "thead" y "tfoot". Cuando este no se define explícitamente, el "tbody" se agrega de forma automática y abarca a todos los elementos)
  • <tfoot> ... </tfoot> (Normalmente contiene el final de la tabla, generalmente resultados)

buenas practicas para tablas

  • Usar "th" para encabezados
  • Agregar el elemento caption
  • Solo usar tablas para información tabular
  • usar "scope" o "id-headers"
  • Aplicar estilos sin perder semántica
  • Solo usar "colspan" y "rowspan" si es necesario
