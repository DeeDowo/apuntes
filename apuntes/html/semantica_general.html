Semántica general HTML

En HTML esencialmente todos los elementos disponibles tienen un significado, es decir, estos tienen una finalidad definida, ya sea para un orden jerárquico o para tipos específicos de información.

Títulos

por defecto existen seis niveles de títulos, desde "h1" hasta "h6", estos sirven para estructurar una página de manera jerárquica. Algunas recomendaciones al usar títulos son las siguientes:

  • De entre todos los títulos de una pagina, el "h1" es el más importante y solo debería de haber uno por página ya que este indica el foco central de la misma.
  • No se deberían usar mas de 3 o 4 niveles de encabezados por página.
  • Usar la jerarquía de los títulos de forma correcta.

Párrafos

la forma mas básica y general de indicar un bloque de texto en un documento HTML.

Elementos dentro de textos

Elementos comunes

<em> </em>
Indica que un bloque o fragmento de texto tiene especial relevancia, en otras palabras, denota la importancia de algo. Da un formato de "itálica".
<strong> </strong>
Resalta que algo (relativo a un texto) tiene mayor importancia. No solo resalta una idea, más bien, indica que esta es de alta relevancia para el texto en particular.
<br>
Fuerza un salto de linea, sin embargo, su uso debería estar limitado a cuando este salto tiene algún significado semántico y no solo como algo visual.
<hr>
Visualmente crea una linea horizontal entre 2 elementos e indica un cambio de tema.

Elementos poco comunes

Citas
Para indicar que un bloque, bloques o lineas de texto son una cita textual o parafraseada existen dos elementos:
  • <blockquote> </blockquote> (Indica que uno o más bloques de texto es o son una cita. Visualmente agrega una apariencia anidada.)
  • <q> </q> (Indica que una linea o lineas de un texto son una cita, agrega un indicador visual de comillas dobles al inicio y fin de la cita.)
Referencias
Existe un elemento y un atributo que acompañan a la citas para indicar la referencia de estas, es decir, el origen de la información.
  • <cite> https... </cite> (le da un valor semántico a la referencia)
  • <q cite="https..."> ... </q> (el atributo cite, se agrega en el elemento para indicar la referencia, sin embargo, este atributo solo agrega metadata, es decir, nada visual)
La forma más común es usar este elemento y atributo en conjunto con un link a la referencia misma.
Abreviaciones y acrónimos
Existe un elemento para indicar que algo es una abreviación o acrónimo de manera semántica, su uso es recomendable solo cuando no se explica la abreviación o acrónimo en el texto circundante.
  • <abbr title="Texto completo de la abreviación/acrónimo"> ... </abbr>
Contacto
Existe un elemento para indicar que algún texto hace referencia a una forma de contacto (no necesariamente una dirección).
  • <address> teléfono: 55... </address>
Superindice y subindice
Existen elementos para indicar que algún texto es un subindice o superindice, su uso más común es con fechas o formulas científicas.
  • <sup> </sup> (ejemplosup)
  • <sub> </sub> (ejemplosub)
Formas de representar código
Para poder mostrar código de forma correcta tanto visual como semántica, tenemos multiples elementos. Estos elementos suelen usarse en conjunto para crear un texto más "rico" en un sentido semántico.
  • <code> </code> (Indica de forma genérica que algo es un código)
  • <pre> </pre> (Muestra el código de forma textual, es decir, no eliminará espacios ni hará algún formateo que no esté explícitamente expresado por nosotros)
  • <var> </var> (Indica que algo es una variable)
  • <kbd> </kbd> (Indica algún "input", generalmente combinaciones de teclas)
  • <samp> </samp> (Indica un "output", es decir, el resultado de algún código)
Fechas y tiempos
Envuelve de manera semántica una fecha, hora o duración de tiempo. Sin embargo, para su uso correcto, requiere del atributo "datetime", el cual asocia una version estandarizada de la fecha y hora en formato "iso 8601". Esto facilita que navegadores, aplicaciones y motores búsqueda puedan interpretar y procesar correctamente la información sin importar el formato o idioma en que se muestra al usuario.
Su estructura es la siguiente:
<time datatime="YYYY-MM-DD-Thh:mm:ssTZD">...</time>
Donde:
  • YYYY: año con cuatro dígitos (por ejemplo, 2025)
  • MM: mes con dos dígitos (por ejemplo, 04 para abril)
  • DD: día con dos dígitos (por ejemplo, 15)
  • T: separador entre la fecha y la hora
  • hh: hora en formato de 24 horas con dos dígitos (por ejemplo, 14 para las 2 PM)
  • mm: minutos con dos dígitos (por ejemplo, 30)
  • ss: segundos con dos dígitos (por ejemplo, 00)
  • TZD: indicador de zona horaria (por ejemplo, Z para UTC o +02:00 para una zona horaria específica)

Listas

existes 3 formas diferentes de indicar listas:

Unordered list
Una lista sin un orden en particular (<ul>...</ul>). Cada elemento de agrega con las etiquetas "list item" (<li>...</li>).
Ordered list
Una lista con un orden en particular (<ol>...</ol>). Cada elemento de agrega con las etiquetas "list item" (<li>...</li>).
Description list
Una lista de descripciones (<dl>...</dl>). A diferencia de "ul" y "ol", para agregar elementos a una lista de descripciones se necesitan dos elementos:
  1. <dt>...</dt> (Definition term)
  2. <dd>...</dd> (Definition description)

Los diferentes tipos de listas pueden anidarse entre si.
