Audio y video

Para la integración de contenido multimedia de tipo audio o video, existen los elementos:

  • <video> </video>
  • <audio> </audio>

Ambos utilizan el atributo "src" para enlazar el contenido:

  • <video src="carpeta/video.formato"> </video>
  • <audio src="carpeta/audio.formato"> </audio>

La principal desventaja del uso del atributo "src" es que de esta forma solo tenemos una posible fuente, lo recomendable y la practica común es utilizar multiples elementos <source> para tener la mayor compatibilidad posible. Cada uno enlazando con un formato diferente y con un atributo "type" que sirve para indicar el formato del archivo, de esta forma le damos multiples opciones al navegador para elegir aquel formato soportado correctamente.

Cuando el atributo "type" no está, el navegador tratará de cargar todos los archivos, uno por uno hasta encontrar uno valido. Cuando sí se utiliza "type", el navegador solo buscará aquel que tenga un formato soportado.

Una buena practica es el agregar un "fallback", es decir, en caso de que ocurra un error o que ningún formato sea valido, poder proporcionar alguna otra opción de acceso al contenido, ya sea dar una alternativa de descarga o al menos poder informar al usuario de que ha ocurrido un error.

Atributos comunes para multimedia
Atributo Función Tipo de multimedia
controls (atributo boolean) Muestra los controles por defecto del navegador para la reproducción del contenido. Audio y Video
width/height Para asignar un tamaño fijo al contenedor, no modifica el "aspect ratio". Video
autoplay (atributo boolean) Hace que el contenido se empiece a reproducir de forma automática al ingresar a la página (no recomendable). Audio y Video
loop (atributo boolean) Hace que el contenido se repita automáticamente. Audio y Video
muted (atributo boolean) Hace que el contenido no tenga sonido por defecto (volumen). Audio y Video
poster Permite agregar una "miniatura". Videos
preload Define el tipo de pre carga del contenido, puede tomar tres valores:
  • auto (pre carga el contenido)
  • metadata (solo pre carga la metadata)
  • none (no pre carga nada)

Algo a tener en consideración es que no es buena practica usar "autoplay" por defecto, siempre es mejor dejar que sea el usuario el que elija si reproducir el contenido o no, especialmente con audios.

Pistas de texto

Para mejorar la accesibilidad y la experiencia del usuario, es posible agregar pistas de texto como subtítulos, descripciones o captions a videos utilizando <track> (elemento void).

El elemento <track> utiliza un archivo en formato ".vtt", este es el formato estándar para mostrar texto en determinados tiempos y por determinados lapsos.

Cada elemento "track" solo puede contener una pista y ciertos atributos para definir el comportamiento de esta:

  • src: Ruta del archivo ".vtt".
  • kind: Tipo de pista.
    • kind="subtitles": Traducción de dialogo (para quienes no hablan el idioma).
    • kind="captions": Transcripción del audio incluyendo efectos sonoros (para sordos).
    • kind="descriptions": Descripción textual del contenido visual.
    • kind="chapter": Navegación por secciones del video.
    • kind="metadata": Información para scripts (no visible para el usuario).
  • srclang: Idioma del archivo.
  • label: Etiqueta visible para el usuario.

Algunas recomendaciones

  • Siempre proveer subtítulos para videos con dialogo
  • Incluir transcripciones para contenido de audio
  • Evitar el uso de "autoplay" o si es necesario, usarlo en conjunto con "muted"
  • Comprimir archivos multimedia adecuadamente
  • Usar el formato más eficiente (webm para video, opus para audio)
  • Considerar usar "streaming" para videos largos
  • Considerar los derechos de autor para cualquier tipo de archivo multimedia
